Purpose
A repeatable system to determine: migrate vs. retain vs. retire — and if migrating, whether to rehost, replatform, refactor, or repurchase.
EAMME v1.0
Decision Framework Expanded 7R
  • Rehost — lift & shift VMs / minimal change
  • Replatform — move to managed services w/ limited code change
  • Refactor — modernization (containers, microservices, serverless)
  • Repurchase — vendor SaaS replacement
  • Retain — keep on‑prem temporarily due to blockers
  • Retire — decommission / archive
How to use Per application
  • Fill out Intake (business + tech + infra + constraints).
  • Assign 1–5 ratings for readiness factors in Ratings.
  • Click Run Evaluation → get route + cloud target + step-by-step plan.
  • Export JSON to build a migration backlog / portfolio plan.
Readiness Score (0–100)
  • 80–100 → Refactor / Modernize
  • 60–79 → Replatform
  • 40–59 → Rehost
  • 20–39 → Retain
  • 0–19 → Retire
Note: Hard gates override score (e.g., obsolete → Retire, SaaS available → Repurchase, hardware/mainframe → Retain).

Tip: Use your intake to anchor ratings:
  • Cloud Compatibility: OS support, networking patterns, dependencies, licensing portability
  • Technical Debt: age, patch posture, brittle releases, manual operations
  • Vendor Lock Risk: proprietary middleware, closed formats, contracts
  • Compliance Complexity: CJIS/FedRAMP/ITAR constraints and audit overhead
  • Modularity: clear components, APIs, separation of concerns
  • Refactor Effort: effort/time to modernize safely
